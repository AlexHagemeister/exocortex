# ISSUES — friction log (open-issue index)

One file per issue in `.state/issues/` (`<YYYY-MM-DD>-<slug>.md`), frontmatter `title` / `status: open|resolved|wontfix` / `opened` / `rule` (what it implicates) / `resolved_by` (filled on close); body freeform, no ceremony — a flag that takes effort won't get filed. To file: create the file, regenerate this index. To close: flip `status`, fill `resolved_by`, regenerate. Resolved files stay on disk as greppable history; this index lists **open issues only** and is derived — never hand-maintained. The weekly digest surfaces open issues; ~3 open issues sharing a `rule` trigger an amend proposal.
